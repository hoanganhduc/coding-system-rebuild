# Claim-Preserving Writing

Use this guidance when revising or drafting prose where wording changes could
change the argument.

## Claim Layer

Before rewriting, extract the draft into atomic claims. Prefer small claims that
can be supported, weakened, or removed independently.

Recommended claim types:

- `contribution`
- `evidence`
- `assumption`
- `caveat`
- `comparison`
- `recommendation`
- `definition`
- `result`
- `limitation`
- `transition`

Recommended support types:

- `source`
- `experiment`
- `theorem`
- `data`
- `author-note`
- `prior-section`
- `missing`

## Rewrite Discipline

- Preserve the claim ledger unless the user asks to change it.
- Improve order, clarity, rhythm, and concision without adding new claims.
- Keep caveats visible when they bound the claim.
- Downgrade overstrong claims when support is partial.
- Flag unsupported claims instead of making them sound more certain.

## Revision Audit

After a rewrite, compare the old and new claim layers.

Track:

- added claims
- removed claims
- strengthened claims
- weakened claims
- caveat changes
- unsupported claims introduced by the rewrite
- evidence mappings that changed

## Readiness Gate

A draft is ready only when:

- substantive claims have visible support or are labeled as unsupported
- unsupported additions from rewrites are removed or explicitly accepted
- caveats and limitations still match the intended argument
- the user can see what changed at the claim level

If material checks remain undone, mark the result `incomplete analysis` and list
what remains unchecked.
